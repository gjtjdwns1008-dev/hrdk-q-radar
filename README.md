# HRDK Q-RADAR — 국가기술자격 법령 통합 레이더

> 매일 새로 시행되는 대한민국 법령을 AI가 읽고, **국가기술자격과의 연관도·노동시장 활용도·자격자 우대 여부**를
> 한 번에 분석해 구글시트 통합 대장과 SQLite 지식베이스에 쌓는 자동화 시스템.
> (1세대 law-monitor + 2세대 HRDK-LAW-RADAR를 통합한 3세대 — 두 시스템 운영에서 얻은 교훈의 집대성)

## 하루의 흐름

```
매시 :20/:50 (GitHub Actions, 하루 48회 시도)
  └ main.py
      ① 시트 → SQLite 재구축 (시트=원본, db=검색 사본)
      ② 자격명칭최신화 반영 (별칭 주입 + 대장 소급)
      ③ 밀린 날짜 계산 → 법제처 연결 확인
      │    └ 실패 → 총괄현황표에 🔴만 기록하고 종료 (숫자 불가침)
      ④ 날짜별: 수집 → 통합 brain 분석(1회 호출) → 우대법령만 워크넷·하이브리드
      ⑤ 통합 대장 24칸 Upsert(MST_ID) → SQLite 저장 → 엑셀+웹훅 → 보류목록
```

성공하면 그날은 끝 — 이후 크론들은 "밀린 날짜 없음"으로 무해 통과합니다.
연결이 계속 막혀도 다음 날 자동 백필로 따라잡습니다(구멍 없음).

## 아홉 가지 설계 원칙 (두 시스템 반년 운영의 교훈)

1. **상태는 항상, 숫자는 성공 시에만** — 실패(🔴)도 총괄현황표에 남긴다. 단 건수 칸은 성공한 실행만 쓴다. (실패가 침묵하면 "차단이 줄었다" 착시가 생기고, 실패가 숫자를 쓰면 0건 오보고가 생긴다 — 둘 다 실제로 겪음)
2. **0건 오보고 차단 + 패자부활전 + 다중 크론** — 3중 방어망. 외부 서버는 반드시 가끔 죽는다.
3. **자격명칭최신화 탭 = 명칭 변경의 단일 진실 소스** — 폐지돼도 삭제하지 않는다(법적 효력 유지). 교체만 한다. "적용여부=완료"로 1회성 보장.
4. **포괄명칭 확장** — 법령이 '미용사'라 쓰면 사전의 미용사(일반)·(피부)·(네일)·(메이크업) 전부로 확장 (core).
5. **값 자동 교정 방어망** — AI를 믿되 검증한다: '대폭증가'→'대폭 증가', Z↔X 짝 강제, 해당없음+우대O 모순 차단.
6. **MST_ID Upsert (자연키 = 법령명|시행일자)** — 같은 법의 재분석은 갱신, 재개정은 새 행.
7. **미리보기 우선** — 시트를 바꾸는 도구는 반드시 엑셀 미리보기 → 사람 확인 → 반영.
8. **웹훅 비우면 발송 안 함** — WEBHOOK_URL 시크릿 미등록 = 메일 0통 (새도우 런의 안전핀).
9. **여기만 고치세요** — 설정은 config 상단에 모은다. 후임자가 코드를 몰라도 운영 가능해야 한다.

## 처음 설치 (신규 담당자용)

1. **구글시트 준비**: 새 스프레드시트 생성 → 서비스계정 이메일을 편집자로 공유 (탭은 코드가 자동 생성)
2. **GitHub Secrets 5종 등록** (Settings→Secrets→Actions):
   `LAW_API_KEY` / `GEMINI_API_KEY` / `GCP_SERVICE_ACCOUNT_JSON`(키 파일 내용 통째) / `GOOGLE_SHEET_URL`(시트 키 또는 URL) / `WORKNET_API_KEY`
   ※ `WEBHOOK_URL`은 **새도우 런 동안 등록하지 않음** — 컷오버 때 등록
3. **기존 데이터 이관** (로컬 PC, 1회):
   ```bash
   cd migrate_tool
   cp env.example .env   # 세 시트 URL 채우기 + gcp-key.json 배치
   SOURCE=live python run_migration.py          # 미리보기 생성 → 눈으로 확인
   SOURCE=live APPLY=1 python run_migration.py  # 빈 대장에만 반영 + 부속탭 복사 + 이음새 설정
   ```
4. push 후 Actions 탭에서 첫 크론(매시 20/50분) 확인 — 첫 실행은 이관 최신일 **+1일부터** 분석

## 총괄현황표 읽는 법

- `07/03 00:20🔴 → 07/03 00:50🔴 → 07/03 01:50🟢` = 그날 시도 이력이 한 줄에 (🔴=법제처 연결 실패, 주로 해외 IP 차단)
- 건수 4칸(총/연관높음/단순/우대)이 채워져 있으면 그 날은 **성공 실행의 숫자** — 실패는 절대 숫자를 건드리지 않음
- A열은 '시행일자'(분석 대상 날짜), 상태칸 타임스탬프는 '시도한 시각' — 둘이 다르면 백필/수동 실행

## 트러블슈팅

|---|---|

| 종목이 이상하게 잘림 | core 업데이트 필요 — `pip install --upgrade --force-reinstall git+…hrdk-law-core.git` |
