## 1. 한눈 지도 — 레포 하나 = 시스템 전부

```
hrdk-q-radar/
├── main.py ─┬─ 일일 배치 진입점 (법령 수집→AI 분석→시트 기록)
│  config.py ├─ 시스템 이름·통합 대장 24칸·총괄 7칸 정의
│  brain.py  ├─ AI 분석 두뇌 (프롬프트·투트랙 분류, core의 LLM 클라이언트 사용)
│  knowledge.py ─ 지식베이스 로더 (541종목 사전 + hrdk_law.db)
│  report_maker.py ─ 시트 기록(통합 대장 Upsert + 총괄현황표)·엑셀 보고서
├── briefing_maker.py / briefing_prompts.py / briefing_pdf.py
│                └─ 월간 이슈브리핑 (docx + xlsx + PDF 디자인판 v12)
├── core/                ★ 공유 라이브러리 (pip install ./core 로 설치)
│   └── hrdk_law_core/   (아래 3절 모듈표 참조)
├── navigator/
│   └── build_site.py    ★ 대국민 공개 사이트 빌더 v2 (core 미의존·자립형)
├── migrate_tool/        이관·정비 도구 5종 (수동 실행, 아래 4절)
├── hrdk_law.db          지식베이스 SQLite (직능연 정리본 등)
└── .github/workflows/   자동 실행 4종 (아래 2절)
```

## 2. 서브시스템별 "무엇으로 작동하나"

| 서브시스템 | 진입점 | 함께 쓰는 파일 | 읽는 것 | 쓰는 것 | 트리거 |
|---|---|---|---|---|---|
| **일일 배치** | `main.py` | config, brain, knowledge, report_maker + **core 전부** | 법제처 API, 워크넷 API, 총괄현황표(마지막 성공일) | 통합 대장 탭, 총괄현황표, (메일 웹훅) | `daily.yml` 30분 간격 48슬롯 |
| **월간 브리핑** | `briefing_maker.py` | briefing_prompts, briefing_pdf + core(sheets·llm·certs) | 통합 대장 탭, 총괄현황표 | docx+xlsx+PDF (Actions Artifacts), (브리핑 웹훅) | `run-briefing.yml` 매월 2일 |
| **네비게이터** | `navigator/build_site.py` | (단독 — core 미의존) | 통합 대장 탭 **하나만** | `dist/index.html` → GitHub Pages | `build-navigator.yml` 하루 3회(08:30/14/20 KST) |
| **수동 재실행** | `main.py` (MANUAL_DATE) | 일일 배치와 동일 | 〃 | 〃 (mark_done 미호출) | `run-manual.yml` 수동 버튼 |

네비게이터 데이터 규칙(v2): 화면1 = `연관도 ∈ {연관높음, 단순관련}` / 화면2 = 진짜 우대만
(`연관도=우대` 또는 실우대분류 4종). `R_SCOPE=all` 환경변수로 v1처럼 전체 포함 가능.

## 5. 데이터 흐름 한 장

```
법제처 API ─┐                        ┌→ 월간 브리핑(docx·xlsx·PDF) ─ 매월 2일
워크넷 API ─┼→ 일일 배치(main.py) →  구글시트                          
직능연 DB  ─┘   AI 분석(Gemini)      [통합 대장 24칸 + 총괄현황표]      
                                     └→ 네비게이터(Pages 공개) ─ 하루 3회
```
원칙: **구글시트 = 유일한 영구 저장소.** 모든 산출물은 시트에서 읽어 만든다.
